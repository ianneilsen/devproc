## sql injection - checking for signs of sql injection in your logs


	grep UNION, select, concat, with http code 200 

	grep outfile

	os command injection

look for outbound sql injection like an email address where the results are sent

	egrep -i '\bUNION\b|\bSELECT\b|\bCHAR\b' $access_log

	egrep -i '\bUNION\b|\bSELECT\b|\bCHAR\b' $access_log | sed -n 's/\([0-9\.]\+\).*/\1/p' | sort | uniq -coutfile
